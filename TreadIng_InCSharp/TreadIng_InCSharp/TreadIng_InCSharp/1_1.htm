نگاه کلی و مفاهیم ( Overview and Concepts )

 

C# اجرای موازی کد را از طریق چند ریسمانی پشتیبانی میکند. یک نخ یا ریسمان یک مسیر اجرای مستقل است که قادر است به طور همزمان با نخهای دیگر اجرا شود.

یک برنامه C# در یک ریسمان منفرد که توسط CLR و سیستم عامل ساخته میشود اجرا میشود (ریسمان اصلی- main -) و با تولید ریسمانهای اضافی چندریسمانی میشود. در اینجا یک مثال ساده و خروجی آنرا می بینیم :

 

برای تمام مثالها فرض شده است که namespace های زیر import شده است ، مگر آنهایی که مشخصاً در مورد آنها چیز دیگری ذکر شده باشد :

 

using System;

using System.Threading;

 

class ThreadTest {

  static void Main() {

    Thread t = new Thread (WriteY);

    t.Start(); // Run WriteY on the new thread

    while (true) Console.Write ("x"); // Write 'x' forever

  }

  static void WriteY() {

    while (true) Console.Write ("y"); // Write 'y' forever

  }

}

ریسمان اصلی یک ریسمان جدید به نام t میسارد که این ریسمان جدید یک متد که حرف y را مکرراً مینویسد اجرا میکند، ریسمان اصلی نیز به طور همزمان حرف x را مکرراً چاپ میکند.

CLR به هر ریسمان پشته حافظه جدایی که فقط  متعلق به آن ریسمان است اختصاص میدهد بنابراین متغیرهای محلی به صورت مجزا ذخیره میشوند. در مثال بعدی ، یک متد را با متغیر محلی برای آن تعریف میکنیم ، سپس متد را در ریسمان اصلی برنامه و یک ریسمان جدید به طور همزمان صدا میکنیم :

static void Main() {

  new Thread (Go).Start(); // Call Go() on a new thread

  Go(); // Call Go() on the main thread

}

static void Go() {

  // Declare and use a local variable - 'cycles'

  for (int cycles = 0; cycles < 5; cycles++) Console.Write ('?');

}

در هر کدام از پشته های حافظه ریسمانها یک کپی مجزا از متغیر cycles ساخته میشود ، لیکن خروجی همانطور که پیش بینی میشود تعداد 10 علامت پرسش است.

ریسمانها در صورتی که منابع مشترکی مانند یک شیء داشته باشند داده ها را به اشتراک میگذارند. در اینجا یک مثال را می بینیم :

class ThreadTest {

  bool done;

  static void Main() {

    ThreadTest tt = new ThreadTest(); // Create a common instance

    new Thread (tt.Go).Start();

    tt.Go();

  }

  // Note that Go is now an instance method

  void Go() {

    if (!done) { done = true; Console.WriteLine ("Done"); }

  }

}

 

Done

 

از آنجا که هر دو ریسمان متد Go() را در یک نمونه یکسان از یک شیء ThreadTest صدا میزنند ، متغیر done را به اشتراک میگذارند . این امر منجر به این خواهد شد که عبارت Done به جای دو بار تنها یک بار چاپ شود.

 

استفاده از اشیاء static نیز راه دیگری برای به اشتراک گذاشتن داده بین نخهاست . در اینجا مثال قبل را در مورد شیء done این بار به عنوان یک متغیر static بررسی می کنیم :

class ThreadTest {

  static bool done; // Static fields are shared between all threads

  static void Main() {

    new Thread (Go).Start();

    Go();

  }

  static void Go() {

    if (!done) { done = true; Console.WriteLine ("Done"); }

  }

}

 

هر دوی این مثالها مفهوم کلیدی دیگری را بازگو می کنند- اینکه از نظر امنیت ریسمان ، خروجی نامعین است : امکان دارد ( گر چه اینگونه به نظر نمی رسد ) که عبارت “Done” دو بار چاپ شود. اگر ترتیب عبارات درون متد Go() را عوض کنیم ، احتمال اینکه عبارت "Done" دو بار چاپ شود وجود دارد :

static void Go() {

  if (!done) { Console.WriteLine ("Done"); done = true; }

}

 

Done

Done (usually!)

 

این مشکل از آنجا ناشی می شود که یک ریسمان مشغول بررسی شرط عبارت if است و درست در همین لحظه ریسمان دیگر مشغول اجرای عبارت WriteLine است و هنوز فرصت نکرده تا done را true کند.

راه حل رفع چنین مشکلاتی استفاده از یک قفل انحصاری در زمان خواندن و نوشتن اشیاء مشترک است.C# عبارت Lock را برای چنین منظوری تهیه دیده است.

class ThreadSafe {

  static bool done;

  static object locker = new object();

  static void Main() {

    new Thread (Go).Start();

    Go();

  }

  static void Go() {

    lock (locker) {

      if (!done) { Console.WriteLine ("Done"); done = true; }

    }

  }

}

وقتی دو ریسمان بر سر یک lock رقابت و ستیز می کنند ( در اینجا ، شیء Locker ) ، یک ریسمان منتظر می ماند ، یا بلوکه میشود تا زمانی که شیء Lock قابل دسترس شود. در مورد این مثال ، این کار تضمین می کند که تنها یک ریسمان در یک زمان وارد بخش بحرانی میشود، و عبارت "Done" تنها یکبار چاپ می شود. وقتی کدی اینگونه - در مقابل نامعلوم بودن در چند ریسمانی -  محافظت میشود میگوییم آن کد thread-safe است.

توقف یا بلوکه شدن موقت ، برای هماهنگ سازی و همزمانی ریسمانها ضروری است. انتظار برای یک قفل انحصاری تنها یک دلیل بلوکه شدن ریسمان است . اینکه ریسمان بخواهد به طور ارادی متوقف شود یا برای یک دوره زمانی بخوابد نیز از دلایل دیگر بلوکه شدن ریسمان است.

 

Thread.Sleep (TimeSpan.FromSeconds (30)); // Block for 30 seconds

 

یک ریسمان ، همچنین می تواند با صدا زدن متد Join یک ریسمان دیگر در انتظار پایان یافتن کار آن ریسمان بماند

 

Thread t = new Thread (Go); // Assume Go is some static method

t.Start();

Thread.Join (t); // Wait (block) until thread t ends

 

یک ریسمان تا زمانی که بلوکه است ، منابع CPU را مصرف نمی کند.

Mutex

شیء Mutex عملکردی مشابه عبارت Lock در C# را ارائه میدهد ، که همین Mutex را بیشتر زائد جلوه میدهد . یکی از فواید آن این است که میتواند مابین چندین پردازه کار کند ( که باعث میشود دامنه قفل به جای یک برنامه در سطح کامپیوتر باشد یعنی computer-wide Lock rather than an application-wide Lock ) .

 

با آنکه Mutex انصافاً سریع است ، ولی Lock صدها بار سریعتر است . اخذ یک Mutex چند میکروثانیه طول میکشد در حالی که اخذ یک Lock فقط چند ده نانو ثانیه طول می کشد (البته بدون در نظر گرفتن زمان بلوکه) .

 

بوسیله متد WaitOne کلاس Mutex میتوان یک قفل انحصاری تولید کرد و در صورت بروز نزاع برای منبع ، اعمال بلوکه کرد. این قفل انحصاری را سپس میتوان با متد ReleaseMutex آزاد کرد . درست مانند عبارت Lock ، شیء Mutex فقط میتواند توسط ریسمانی که آنرا اخذ کرده است رها شود .

یک کاربرد مرسوم برای یک cross-process Mutex (Mutex فرا پردازه ای) این است که تضمین کنیم در یک زمان فقط یک نمونه از یک برنامه اجرا میشود . در اینجا این را نشان می دهیم :

class OneAtATimePlease {

  // Use a name unique to the application (eg include your company URL)

  static Mutex mutex = new Mutex (false, "oreilly.com OneAtATimeDemo");

  static void Main() {

    // Wait 5 seconds if contended – in case another instance

    // of the program is in the process of shutting down.

    if (!mutex.WaitOne (TimeSpan.FromSeconds (5), false)) {

      Console.WriteLine ("Another instance of the app is running. Bye!");

      return;

    }

    try {

      Console.WriteLine ("Running - press Enter to exit");

      Console.ReadLine();

    }

    finally { mutex.ReleaseMutex(); }

  }

}

یک خوبی Mutex این است که در صورتی که اگر قبل از اینکه متد ReleaseMutex را صدا کنیم ، به آن خاتمه دهیم (terminate) ، CLR به طور اتوماتیک Mutex را آزاد می کند .

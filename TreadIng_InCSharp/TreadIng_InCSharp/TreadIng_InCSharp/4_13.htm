انصراف از نخها Aborting Threads

یک ریسمان میتواند با اعمال زور و از طریق متد Abort پایان یابد .

 

class Abort {

  static void Main() {

    Thread t = new Thread (delegate() {while(true);}); // Spin forever

    t.Start();

    Thread.Sleep (1000); // Let it run for a second...

    t.Abort(); // then abort it.

  }

}

 

ریسمانی که انصراف می خورد بلافاصله وارد حالت AbortRequested میشود . سپس همانطور که انتظار میرود در صورتیکه پایان یابد به حالت Stopped میرود . متد فراخوان میتواند توسط فراخوانی متد Join منتظر پایان یافتن کار ریسمانی که منصرف کرده است بماند :

 

class Abort {

  static void Main() {

    Thread t = new Thread (delegate() { while (true); });

    Console.WriteLine (t.ThreadState); // Unstarted

    t.Start();

    Thread.Sleep (1000);

    Console.WriteLine (t.ThreadState); // Running

    t.Abort();

    Console.WriteLine (t.ThreadState); // AbortRequested

    t.Join();

    Console.WriteLine (t.ThreadState); // Stopped

  }

}

 

فراخوانی متد Abort در اکثر مواقع درست در جای که ریسمان در حال اجراست باعث ایجاد یک استثناء ThreadAbortException در ریسمان هدف ( ریسمانی که انصراف خورده ) می شود . ریسمانی که انصراف خورده میتواند این استثناء را اداره کند ولی دوباره به طور اتوماتیک در انتهای بلاک catch ایجاد میشود . ( برای کمک به اینکه تضمین کند ریسمان ، واقعاً همانطور که انتظار میزود ، پایان یافته است ) . هر چند میتوان از ایجاد دوباره استثناء به طور اتوماتیک توسط فراخوانی متد Thread.ResetAbort در بلاک Catch جلوگیری کرد . با این کار سپس ریسمان دوباره وارد حالت Running میشود ( که در آنجا دوباره قابلیت انصراف دوباره را خواهد داشت ) . در مثال زیر ، ریسمان کارگر هر زمان که یک انصراف میخورد دوباره از مرگ به حیات بر میگردد .

 

class Terminator {

  static void Main() {

    Thread t = new Thread (Work);

    t.Start();

    Thread.Sleep (1000); t.Abort();

    Thread.Sleep (1000); t.Abort();

    Thread.Sleep (1000); t.Abort();

  }

  static void Work() {

    while (true) {

      try { while (true); }

      catch (ThreadAbortException) { Thread.ResetAbort(); }

      Console.WriteLine ("I will not die!");

    }

  }

}

 

استثناء ThreadAbortException خاص زمان اجراست ، رخ دادن آن در صورتیکه اداره نشده باشد باعث نمیشود تا برنامه پایان یابد ، بر خلاف دیگر انواع استثناء ها .

متد Abort تقریباً در تمام حالتها بر یک ریسمان اعمال میشود . running ، blocked ، suspended و stopped . هرچند میتوان انصراف را بر یک ریسمان معلق (suspended) اعمال کرد و یک استثناء ThreadStateException رخ خواهد داد . ( در این زمان در ریسمان فراخوان ) ولی انصراف تا زمانی که ریسمان معلق دوباره از سرگیری (resume) نشود صورت نمیگیرد . در اینجا میبینیم که چگونه یک ریسمان معلق را انصراف بزنیم.

 

try { suspendedThread.Abort(); }

catch (ThreadStateException) { suspendedThread.Resume(); }

// Now the suspendedThread will abort.

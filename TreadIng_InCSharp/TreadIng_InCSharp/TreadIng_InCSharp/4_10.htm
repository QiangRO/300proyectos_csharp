رقابتها و تصدیق Races and Acknowledgement

میخواهیم یک ریسمان کارگر را 5 بار مسلسل وار سیگنال بدهیم :

 

class Race {

  static object locker = new object();

  static bool go;

  static void Main() {

    new Thread (SaySomething).Start();

    for (int i = 0; i < 5; i++) {

      lock (locker) { go = true; Monitor.Pulse (locker); }

    }

  }

  static void SaySomething() {

    for (int i = 0; i < 5; i++) {

      lock (locker) {

        while (!go) Monitor.Wait (locker);

        go = false;

      }

      Console.WriteLine ("Wassup?");

    }

  }

}

 

Expected Output:

 

Wassup?

Wassup?

Wassup?

Wassup?

Wassup?

 

Actual Output:

 

Wassup?

(hangs)

 

 

این برنامه مخرب و ناکارآمد است : حلقه for در ریسمان اصلی برنامه در طول پنج دوره اش هر زمانی که ریسمان کارگر قفل را نگه ندارد می تاند با دنده خلاص حرکت کند .بهیچوجه قبل از شروع ریسمان کارگر ! مثال تولیدکننده/مصرف کننده از مشکلش رنج نمی برد زیرا اگر ریسمان اصلی برنامه از ریسمان کارگر جلوتر بیافتد ، هر درخواستی به سادگی در صف قرار میگیرد . ولی در این مورد احتیاج داریم که ریسمان اصلی برنامه در هر دوره از اجرای حلقه در صورتیکه کارگر هنوز مشغول انجام کار قبلی است ، بلوکه شود .

یک راه حل ساده برای ریسمان اصلی برنامه این است که پس از هر سیکل اجرای حلقه منتظر بماند تا پرچم go توسط ریسمان کارگر پاک شود . برای این کار لازم است تا ریسمان کارگر پس از پاک کردن پرچم go اعمال Pulse کند :

 

class Acknowledged {

  static object locker = new object();

  static bool go;

  static void Main() {

    new Thread (SaySomething).Start();

    for (int i = 0; i < 5; i++) {

      lock (locker) { go = true; Monitor.Pulse (locker); }

      lock (locker) { while (go) Monitor.Wait (locker); }

    }

  }

  static void SaySomething() {

    for (int i = 0; i < 5; i++) {

      lock (locker) {

        while (!go) Monitor.Wait (locker);

        go = false; Monitor.Pulse (locker); // Worker must Pulse

      }

      Console.WriteLine ("Wassup?");

    }

  }

}

 

Wassup? (repeated five times)

 

یک ویژگی مهم برنامهای اینچنینی این است که ریسمان کارگر قبل از اینکه کار گاهاً زمانگیر خود را انجام دهد قفلش را رها می کند ( این اتفاق زمانی می افتد که ما Console.WriteLine را فراخوانی می کنیم ) . این رفتار تضمین می کند که ریسمان محرک هنگامیکه کارگر مشغول انجام کاری که برایش سیگنال گرفته است بی جهت بلوکه نشود . ( و فقط زمانی بلوکه شود که ریسمان کارگر مشغول انجام کار قبلی است ) .

 

در این مثال فقط یک ریسمان ( ریسمان اصلی برنامه ) ریسمان کارگر را برای انجام کار سیگنال میدهد . اگر چندین ریسمان به کارگر سیگنال دهند ( یعنی منطق ریسمان اصلی را استفاده کنند ) شکست خواهیم خورد . دو ریسمان سیگنال دهنده هر کدام میتوانند به ترتیب خط زیر از کد را اجرا کنند :

 

lock (locker) { go = true; Monitor.Pulse (locker); }

 

اگر ریسمان کارگر اولین سیگنال را هنوز پردازش نکرده باشد ، سیگنال دوم گم خواهد شد . میتوانیم طرح خود را در این سناریو با استفاده از یک جفت پرچم پایدار کنیم . یک پرچم "raedy" و همچنین یک پرچم "go" مثل قبل . این مشابه مثال قبلی است که کار مشابهی را با استفاده از دو AutoResetEvents انجام میدهد با این تفاوت که توسعه پذیرتر است . در اینجا الگویی که با استفاده از فیلدهای نمونه دوباره بررسی شده است را می بینیم :

 

Wait/Pulse Boilerplate #3: Two-way Signaling

public class Acknowledged {

  object locker = new object();

  bool ready;

  bool go;

  public void NotifyWhenReady() {

    lock (locker) {

      // Wait if the worker's already busy with a previous job

      while (!ready) Monitor.Wait (locker);

      ready = false;

      go = true;

      Monitor.PulseAll (locker);

    }

  }

  public void AcknowledgedWait() {

    // Indicate that we're ready to process a request

    lock (locker) { ready = true; Monitor.Pulse (locker); }

    lock (locker) {

      while (!go) Monitor.Wait (locker); // Wait for a "go" signal

      go = false; Monitor.PulseAll (locker); // Acknowledge signal

    }

    Console.WriteLine ("Wassup?"); // Perform task

  }

}

 

برای تشریح مساله دو ریسمان همزمان را آغاز خواهیم کرد که هرکدام 5 مرتبه ریسمان کارگر را آگاه میکند (پیام میدهد) . در ضمن ریسمان اصلی هم برای 10 پیام منتظر خواهد شد .

 

 

public class Test {

  static Acknowledged a = new Acknowledged();

  static void Main() {

    new Thread (Notify5).Start(); // Run two concurrent

    new Thread (Notify5).Start(); // notifiers...

    Wait10(); // ... and one waiter.

  }

  static void Notify5() {

    for (int i = 0; i < 5; i++)

    a.NotifyWhenReady();

  }

  static void Wait10() {

    for (int i = 0; i < 10; i++)

    a.AcknowledgedWait();

  }

}

Wassup?

(repeated ten times)

 

 

در متد Notify ، پرچم ready قبل از خروج از عبارت lock پاک میشود . این کار اهمیت حیاتی دارد : این کار از ارسال سیگنال توسط دو ریسمان آگاه کننده به طور متوالی بدون چک کردن مجدد پرچم جلوگیری می کند ، به منظور سادگی پرچم go و فراخوانی PulseAll را در یک عبارت lock انجام داده ایم گرچه میتوانستیم آنها را در عبارات lock جداگانه ای بگذاریم و چیزی تغییر نمیکرد .
